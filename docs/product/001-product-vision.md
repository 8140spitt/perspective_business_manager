# 001 Product vision

PBM is a business-first enterprise platform.

It gives a business one connected place to define itself, manage people, win work, deliver work, control money, manage suppliers, maintain evidence, govern risk and report performance.

## Product language

PBM uses clear product language:

- Workspaces
- Capabilities
- Activities
- Business objects
- Data spine
- Integrated records

PBM is not organised around old module labels. The user should know where to go because of the job they are doing.

## Core idea

People do work for clients.

Work happens inside projects, services, instructions, operational activities and controlled processes.

That work creates value, cost, evidence, risk and performance data.

PBM connects those records so the business can see the truth.

## Design test

A PBM screen is correct when a user can answer:

1. What job am I doing here?
2. Which business object am I working with?
3. Which workspace owns this activity view?
4. Which other workspaces use the same record?
5. What is the single source of truth underneath?
